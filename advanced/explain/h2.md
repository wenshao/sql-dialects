> - ============================================================
> - EXPLAIN 基本用法
> - ============================================================

```sql
EXPLAIN SELECT * FROM users WHERE username = 'alice';
```

输出示例：
SELECT "USERS"."ID", "USERS"."USERNAME", ...
FROM "PUBLIC"."USERS"
/* PUBLIC.IDX_USERS_USERNAME: USERNAME = 'alice' */
WHERE "USERS"."USERNAME" = 'alice'

## EXPLAIN ANALYZE（实际执行）


```sql
EXPLAIN ANALYZE SELECT * FROM users WHERE age > 25;
```

## 输出包含实际执行统计：

/* scanCount: 100 */

## EXPLAIN 用于 DML


```sql
EXPLAIN INSERT INTO users (username, email) VALUES ('test', 'test@example.com');
EXPLAIN UPDATE users SET age = 30 WHERE username = 'alice';
EXPLAIN DELETE FROM users WHERE age < 18;
```

## 执行计划中的注释含义


/* PUBLIC.PRIMARY_KEY_2: ID = 1 */        通过主键查找
/* PUBLIC.IDX_USERS_AGE: AGE > 25 */      通过索引范围扫描
/* PUBLIC.USERS.tableScan */               全表扫描
/* scanCount: 100 */                       扫描行数（EXPLAIN ANALYZE）

## 查看表的行数和磁盘使用

```sql
SELECT TABLE_NAME, ROW_COUNT_ESTIMATE, DISK_SPACE_USED
FROM INFORMATION_SCHEMA.TABLES
WHERE TABLE_SCHEMA = 'PUBLIC';
```

## 查看索引信息

```sql
SELECT TABLE_NAME, INDEX_NAME, INDEX_TYPE_NAME, COLUMN_NAME
FROM INFORMATION_SCHEMA.INDEXES
WHERE TABLE_SCHEMA = 'PUBLIC';
```

## 启用跟踪（输出到文件）

```sql
SET TRACE_LEVEL_FILE 3;   -- 0=OFF, 1=ERROR, 2=INFO, 3=DEBUG
SET TRACE_LEVEL_SYSTEM_OUT 2;
```

## H2 连接字符串中启用跟踪：

jdbc:h2:~/test;TRACE_LEVEL_FILE=3

## 更新统计信息

```sql
ANALYZE;
ANALYZE TABLE users;
```

注意：H2 的 EXPLAIN 输出包含优化后的 SQL 和使用的索引
注意：EXPLAIN ANALYZE 会实际执行查询并输出 scanCount
注意：H2 作为嵌入式数据库，执行计划相对简单
注意：跟踪级别可以通过 SET TRACE_LEVEL 控制
注意：ANALYZE 收集统计信息供优化器使用
