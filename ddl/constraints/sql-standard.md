# SQL 标准: 约束

> 参考资料:
> - [ISO/IEC 9075 SQL Standard](https://www.iso.org/standard/76583.html)
> - [Modern SQL - by Markus Winand](https://modern-sql.com/)
> - [Modern SQL - Constraints](https://modern-sql.com/concept/constraint)

## SQL-86 (SQL-1): 最初的标准

仅支持 NOT NULL

```sql
CREATE TABLE users (
    id       INTEGER NOT NULL,
    username CHAR(64) NOT NULL,
    email    CHAR(255)
);
```

## SQL-89 (SQL-1, 修正版): 引入完整性约束

新增 PRIMARY KEY, UNIQUE, FOREIGN KEY, CHECK, DEFAULT

```sql
CREATE TABLE users (
    id       INTEGER NOT NULL,
    username CHAR(64) NOT NULL,
    email    CHAR(255),
    age      INTEGER,
    PRIMARY KEY (id),
    UNIQUE (username),
    CHECK (age >= 0)
);
```

```sql
CREATE TABLE orders (
    id      INTEGER NOT NULL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id)
);
```

## SQL-92 (SQL2): 完整的约束体系

新增命名约束（CONSTRAINT name）
新增引用动作（CASCADE, SET NULL, SET DEFAULT, RESTRICT, NO ACTION）
新增 DEFERRABLE 约束
新增 DOMAIN（带约束的自定义类型）

```sql
CREATE TABLE users (
    id       INTEGER NOT NULL,
    username VARCHAR(64) NOT NULL,
    email    VARCHAR(255),
    age      INTEGER DEFAULT 0,
    CONSTRAINT pk_users PRIMARY KEY (id),
    CONSTRAINT uk_username UNIQUE (username),
    CONSTRAINT chk_age CHECK (age >= 0 AND age <= 200)
);
```

```sql
CREATE TABLE orders (
    id      INTEGER NOT NULL,
    user_id INTEGER NOT NULL,
    CONSTRAINT pk_orders PRIMARY KEY (id),
    CONSTRAINT fk_orders_user FOREIGN KEY (user_id) REFERENCES users (id)
        ON DELETE CASCADE
        ON UPDATE CASCADE
);
```

可延迟约束（在事务提交时再校验）
```sql
CREATE TABLE t (
    id  INTEGER PRIMARY KEY,
    ref INTEGER,
    CONSTRAINT fk_self FOREIGN KEY (ref) REFERENCES t (id)
        DEFERRABLE INITIALLY DEFERRED
);
```

DOMAIN
```sql
CREATE DOMAIN positive_int AS INTEGER
    CHECK (VALUE > 0);
```

```sql
CREATE DOMAIN email_type AS VARCHAR(255)
    CHECK (VALUE LIKE '%@%.%');
```

## SQL:1999 (SQL3): ASSERTION

新增 ASSERTION（跨表的约束，几乎没有数据库实现）

```sql
CREATE ASSERTION chk_total_balance
    CHECK ((SELECT SUM(balance) FROM accounts) >= 0);
```

- **注意：ASSERTION 在标准中定义但几乎无实现**

## SQL:2003: IDENTITY 约束

IDENTITY 列隐含 NOT NULL

```sql
CREATE TABLE users (
    id       BIGINT GENERATED ALWAYS AS IDENTITY,
    username VARCHAR(64) NOT NULL,
    CONSTRAINT pk_users PRIMARY KEY (id)
);
```

GENERATED ALWAYS: 不允许手动插入值
GENERATED BY DEFAULT: 允许手动插入值

## SQL:2011: 时态约束

时态表的 PERIOD 约束

```sql
CREATE TABLE contracts (
    id         BIGINT PRIMARY KEY,
    start_date DATE,
    end_date   DATE,
    PERIOD FOR valid_period (start_date, end_date),
    CHECK (end_date > start_date)
);
```

## 约束的标准分类

列级约束: NOT NULL, DEFAULT, PRIMARY KEY, UNIQUE, CHECK, REFERENCES
表级约束: PRIMARY KEY, UNIQUE, CHECK, FOREIGN KEY
断言约束: ASSERTION（跨表，几乎无实现）

约束动作:
RESTRICT: 如果有依赖则拒绝操作（立即检查）
NO ACTION: 如果有依赖则拒绝操作（可延迟检查，默认）
CASCADE: 级联操作
SET NULL: 设置为 NULL
SET DEFAULT: 设置为默认值

- **注意：没有数据库完全实现所有标准约束**
- **注意：ASSERTION 定义了20多年，几乎没有数据库实现**
- **注意：DEFERRABLE 约束只有 PostgreSQL 和 Oracle 完整实现**
- **注意：CHECK 约束 MySQL 直到 8.0.16 才真正执行**
