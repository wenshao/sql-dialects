# SQL 标准: CREATE TABLE

> 参考资料:
> - [ISO/IEC 9075 SQL Standard](https://www.iso.org/standard/76583.html)
> - [Modern SQL - by Markus Winand](https://modern-sql.com/)
> - [Modern SQL - CREATE TABLE](https://modern-sql.com/feature/create-table)

## SQL-86 (SQL-1): 最初的标准

基本的 CREATE TABLE，支持列定义、NOT NULL
```sql
CREATE TABLE users (
    id       INTEGER NOT NULL,
    username CHAR(64) NOT NULL,
    email    CHAR(255)
);
```

## SQL-89 (SQL-1, 修正版): 增加完整性约束

新增 PRIMARY KEY, UNIQUE, FOREIGN KEY, CHECK, DEFAULT
```sql
CREATE TABLE users (
    id       INTEGER NOT NULL,
    username CHAR(64) NOT NULL,
    email    CHAR(255),
    PRIMARY KEY (id),
    UNIQUE (username),
    CHECK (id > 0)
);
```

```sql
CREATE TABLE orders (
    id      INTEGER NOT NULL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id)   -- 外键
);
```

## SQL-92 (SQL2): 重大扩展

新增 VARCHAR, NUMERIC, DECIMAL, DATE, TIME, TIMESTAMP, INTERVAL
新增 TEMPORARY TABLE
新增 DOMAIN（用户自定义类型的前身）
新增 CASCADE / RESTRICT 引用动作

```sql
CREATE TABLE users (
    id         INTEGER NOT NULL,
    username   VARCHAR(64) NOT NULL,
    email      VARCHAR(255),
    age        INTEGER,
    balance    DECIMAL(10,2) DEFAULT 0.00,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    UNIQUE (username),
    CHECK (age >= 0)
);
```

```sql
CREATE TABLE orders (
    id      INTEGER NOT NULL PRIMARY KEY,
    user_id INTEGER NOT NULL,
    amount  DECIMAL(10,2),
    FOREIGN KEY (user_id) REFERENCES users(id)
        ON DELETE CASCADE
        ON UPDATE CASCADE
);
```

```sql
CREATE GLOBAL TEMPORARY TABLE temp_data (
    id   INTEGER,
    data VARCHAR(255)
) ON COMMIT DELETE ROWS;
```

创建 DOMAIN（约束的抽象）
```sql
CREATE DOMAIN email_type AS VARCHAR(255)
    CHECK (VALUE LIKE '%@%.%');
```

## SQL:1999 (SQL3): 面向对象扩展

新增 BOOLEAN 类型
新增 LOB 类型：BLOB, CLOB
新增 ARRAY 类型
新增 用户自定义类型（UDT）
新增 GENERATED（计算列）
新增 触发器在标准中定义

```sql
CREATE TABLE users (
    id         INTEGER NOT NULL,
    username   VARCHAR(64) NOT NULL,
    email      VARCHAR(255),
    active     BOOLEAN DEFAULT TRUE,
    bio        CLOB,
    avatar     BLOB,
    tags       VARCHAR(32) ARRAY[10],       -- 数组类型
    PRIMARY KEY (id)
);
```

## SQL:2003: 自增和窗口函数

新增 IDENTITY 列（标准自增方式）
新增 BIGINT 类型
新增 MULTISET 类型
新增 MERGE 语句
窗口函数正式纳入标准

```sql
CREATE TABLE users (
    id         BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    username   VARCHAR(64) NOT NULL UNIQUE,
    email      VARCHAR(255),
    full_name  VARCHAR(129) GENERATED ALWAYS AS (username || '@' || email)  -- 计算列
);
```

IDENTITY 变体：
GENERATED ALWAYS AS IDENTITY: 总是自动生成
GENERATED BY DEFAULT AS IDENTITY: 可以手动指定

## SQL:2008: TRUNCATE 和增强

新增 TRUNCATE TABLE
INSTEAD OF 触发器
增强 MERGE 语句

```sql
TRUNCATE TABLE users;
```

## SQL:2011: 时态表

新增系统版本化时态表（System-Versioned Temporal Table）
新增应用时态表（Application-Time Period Table）

```sql
CREATE TABLE users (
    id         BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    username   VARCHAR(64) NOT NULL,
    valid_from TIMESTAMP(6) GENERATED ALWAYS AS ROW START,
    valid_to   TIMESTAMP(6) GENERATED ALWAYS AS ROW END,
    PERIOD FOR SYSTEM_TIME (valid_from, valid_to)
) WITH SYSTEM VERSIONING;
```

应用时态（业务时间）
```sql
CREATE TABLE contracts (
    id         BIGINT PRIMARY KEY,
    start_date DATE,
    end_date   DATE,
    PERIOD FOR valid_period (start_date, end_date)
);
```

## SQL:2016: JSON 和多态表函数

新增 JSON 支持（JSON_VALUE, JSON_QUERY, JSON_TABLE 等）
新增 行模式识别（MATCH_RECOGNIZE）
新增 LISTAGG

JSON 列没有标准类型关键字，建议用 VARCHAR 或实现自定义
```sql
CREATE TABLE events (
    id   BIGINT PRIMARY KEY,
    data VARCHAR(4000)  -- 存储 JSON，用 IS JSON 校验
    CHECK (data IS JSON)
);
```

## SQL:2023: 最新标准

新增 ANY_VALUE 聚合函数
新增 GREATEST / LEAST
新增 数字字面量分隔符（1_000_000）
增强 JSON 支持
增强 图查询（SQL/PGQ）

- **注意：没有数据库完全实现了任何一个 SQL 标准**
各数据库通常实现标准的子集，再加上自己的扩展
