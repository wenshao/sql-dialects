## SEQUENCE（兼容 PostgreSQL）

```sql
CREATE SEQUENCE user_id_seq
    START WITH 1
    INCREMENT BY 1
    NO MINVALUE
    NO MAXVALUE
    CACHE 20;                                -- Greenplum 中 CACHE 很重要，减少 Coordinator 协调
```


使用序列
```sql
INSERT INTO users (id, username) VALUES (nextval('user_id_seq'), 'alice');
SELECT nextval('user_id_seq');
SELECT currval('user_id_seq');
SELECT setval('user_id_seq', 1000);
```


修改序列
```sql
ALTER SEQUENCE user_id_seq RESTART WITH 1000;
ALTER SEQUENCE user_id_seq CACHE 100;
```


删除序列
```sql
DROP SEQUENCE user_id_seq;
DROP SEQUENCE IF EXISTS user_id_seq;
```


## SERIAL / BIGSERIAL

```sql
CREATE TABLE users (
    id       BIGSERIAL PRIMARY KEY,
    username VARCHAR(64) NOT NULL,
    email    VARCHAR(255) NOT NULL
) DISTRIBUTED BY (id);
```


SERIAL 等价于：
CREATE SEQUENCE users_id_seq;
CREATE TABLE users (id BIGINT DEFAULT nextval('users_id_seq') ...);

## GENERATED AS IDENTITY（Greenplum 7+ / PostgreSQL 10+ 兼容）

```sql
CREATE TABLE orders (
    id       BIGINT GENERATED ALWAYS AS IDENTITY,
    amount   DECIMAL(10,2)
) DISTRIBUTED BY (id);

CREATE TABLE products (
    id       BIGINT GENERATED BY DEFAULT AS IDENTITY (START WITH 100 INCREMENT BY 1),
    name     VARCHAR(64)
) DISTRIBUTED BY (id);
```


## UUID 生成

需要 uuid-ossp 扩展
```sql
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
SELECT uuid_generate_v4();

CREATE TABLE sessions (
    id         UUID DEFAULT uuid_generate_v4(),
    user_id    BIGINT,
    created_at TIMESTAMP DEFAULT current_timestamp
) DISTRIBUTED BY (id);
```


## 分布式环境下的序列注意事项

1. 序列在 Coordinator 节点生成，可能成为瓶颈
2. 使用 CACHE 减少 Coordinator 协调次数
3. UUID 在分布式环境中更高效（各 Segment 独立生成）
4. DISTRIBUTED BY 分布键选择影响数据均匀性

序列 vs 自增 权衡：
1. SERIAL/BIGSERIAL：简单，兼容 PostgreSQL
2. SEQUENCE + CACHE：性能优化的序列
3. UUID：分布式最佳，无 Coordinator 瓶颈
4. GENERATED AS IDENTITY（7+）：SQL 标准兼容
