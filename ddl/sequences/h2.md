> - SEQUENCE
> - ============================================

```sql
CREATE SEQUENCE user_id_seq;

CREATE SEQUENCE order_id_seq
    START WITH 1
    INCREMENT BY 1
    MINVALUE 1
    MAXVALUE 9223372036854775807
    CACHE 32
    NO CYCLE;
```

## 使用序列

```sql
INSERT INTO users (id, username) VALUES (NEXT VALUE FOR user_id_seq, 'alice');
SELECT NEXT VALUE FOR user_id_seq;
SELECT CURRENT VALUE FOR user_id_seq;
```

## 修改序列

```sql
ALTER SEQUENCE user_id_seq RESTART WITH 1000;
ALTER SEQUENCE user_id_seq INCREMENT BY 2;
```

## 删除序列

```sql
DROP SEQUENCE user_id_seq;
DROP SEQUENCE IF EXISTS user_id_seq;
```

## IDENTITY 列

## GENERATED ALWAYS AS IDENTITY（推荐，H2 2.0+）

```sql
CREATE TABLE users (
    id       BIGINT GENERATED ALWAYS AS IDENTITY,
    username VARCHAR(64) NOT NULL,
    email    VARCHAR(255) NOT NULL
);
```

## GENERATED BY DEFAULT AS IDENTITY

```sql
CREATE TABLE orders (
    id       BIGINT GENERATED BY DEFAULT AS IDENTITY (START WITH 100),
    amount   DECIMAL(10,2)
);
```

## 传统 AUTO_INCREMENT 语法（H2 兼容模式）

```sql
CREATE TABLE legacy_users (
    id       BIGINT AUTO_INCREMENT,
    username VARCHAR(64)
);
```

## IDENTITY 类型（H2 特有的简写）

```sql
CREATE TABLE items (
    id       IDENTITY,                       -- 等价于 BIGINT GENERATED BY DEFAULT AS IDENTITY
    name     VARCHAR(64)
);
```

## UUID 生成

```sql
SELECT RANDOM_UUID();
SELECT UUID();

CREATE TABLE sessions (
    id         UUID DEFAULT RANDOM_UUID(),
    user_id    BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 序列 vs 自增 权衡

## GENERATED AS IDENTITY（推荐）：SQL 标准，H2 2.0+

## IDENTITY 类型：H2 特有简写，方便快捷

## SEQUENCE：灵活，可跨表共享

## AUTO_INCREMENT：兼容 MySQL 模式

## RANDOM_UUID()：全局唯一标识
