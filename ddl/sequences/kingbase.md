> - ============================================
> - SEQUENCE（兼容 PostgreSQL）
> - ============================================

```sql
CREATE SEQUENCE user_id_seq
    START WITH 1
    INCREMENT BY 1
    NO MINVALUE
    NO MAXVALUE
    CACHE 20;
```

## 使用序列

```sql
INSERT INTO users (id, username) VALUES (nextval('user_id_seq'), 'alice');
SELECT nextval('user_id_seq');
SELECT currval('user_id_seq');
SELECT setval('user_id_seq', 1000);
```

## 修改序列

```sql
ALTER SEQUENCE user_id_seq RESTART WITH 1000;
```

## 删除序列

```sql
DROP SEQUENCE user_id_seq;
DROP SEQUENCE IF EXISTS user_id_seq;
```

## SERIAL / BIGSERIAL

```sql
CREATE TABLE users (
    id       BIGSERIAL PRIMARY KEY,
    username VARCHAR(64) NOT NULL,
    email    VARCHAR(255) NOT NULL
);
```

## GENERATED AS IDENTITY

```sql
CREATE TABLE orders (
    id       BIGINT GENERATED ALWAYS AS IDENTITY,
    amount   DECIMAL(10,2)
);

CREATE TABLE products (
    id       BIGINT GENERATED BY DEFAULT AS IDENTITY (START WITH 100 INCREMENT BY 1),
    name     VARCHAR(64)
);
```

## Oracle 兼容模式

KingbaseES 在 Oracle 兼容模式下支持 Oracle 风格的序列

CREATE SEQUENCE user_id_seq START WITH 1 INCREMENT BY 1;
SELECT user_id_seq.NEXTVAL FROM DUAL;
SELECT user_id_seq.CURRVAL FROM DUAL;

## UUID 生成

## 需要 uuid-ossp 扩展

```sql
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
SELECT uuid_generate_v4();

SELECT sys_guid();                           -- KingbaseES 兼容 Oracle 的 SYS_GUID
```

## 序列 vs 自增 权衡

## SERIAL/BIGSERIAL：简单，兼容 PostgreSQL

## GENERATED AS IDENTITY：SQL 标准兼容

## SEQUENCE：灵活，可跨表共享

## UUID：全局唯一

## KingbaseES 同时支持 PostgreSQL 和 Oracle 两种语法风格
