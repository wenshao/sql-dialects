## IDENTITY 列

Redshift 使用 IDENTITY 代替 SEQUENCE
```sql
CREATE TABLE users (
    id       BIGINT IDENTITY(1, 1),          -- IDENTITY(seed, step)
    username VARCHAR(64) NOT NULL,
    email    VARCHAR(255) NOT NULL
) DISTSTYLE KEY DISTKEY(id) SORTKEY(id);
```


IDENTITY(seed, step)
```sql
CREATE TABLE orders (
    id       BIGINT IDENTITY(1000, 1),
    amount   DECIMAL(10,2)
);
```


DEFAULT IDENTITY（允许手动指定值）
```sql
CREATE TABLE products (
    id       BIGINT DEFAULT IDENTITY(1, 1),
    name     VARCHAR(64)
);
```

IDENTITY：不允许手动指定
DEFAULT IDENTITY：允许手动指定（如果未指定则自动生成）

## GENERATED AS IDENTITY（Redshift 较新版本）

```sql
CREATE TABLE logs (
    id       BIGINT GENERATED BY DEFAULT AS IDENTITY(1, 1),
    message  VARCHAR(1000)
);
```


## Redshift 不支持 CREATE SEQUENCE

替代方案：使用 ROW_NUMBER()
```sql
SELECT
    ROW_NUMBER() OVER (ORDER BY created_at) AS row_id,
    username,
    email
FROM users;
```


模拟序列（使用临时表和 IDENTITY）
```sql
CREATE TEMP TABLE seq_temp (
    seq_val BIGINT IDENTITY(1,1),
    dummy   CHAR(1)
);
```


## UUID 生成（无内置 UUID 函数）

替代方案 1：使用 MD5 + 随机数 + 时间戳
```sql
SELECT MD5(RANDOM()::TEXT || GETDATE()::TEXT) AS pseudo_uuid;
```


替代方案 2：使用 Python UDF
CREATE FUNCTION f_uuid()
RETURNS VARCHAR(36)
STABLE
AS $$ import uuid; return str(uuid.uuid4()) $$
LANGUAGE plpythonu;

## 序列 vs 自增 权衡

1. IDENTITY（推荐）：Redshift 原生支持
2. DEFAULT IDENTITY：允许手动指定，适合数据迁移
3. ROW_NUMBER()：查询时编号
4. MD5 模拟 UUID：适合需要唯一标识的场景
5. IDENTITY 值在并发 COPY 时可能不连续

限制：
不支持 CREATE SEQUENCE
不支持 SERIAL / BIGSERIAL
不支持内置 UUID 函数
IDENTITY 列不支持 UPDATE
一张表只能有一个 IDENTITY 列
