## SEQUENCE

```sql
CREATE SEQUENCE user_id_seq
    START WITH 1
    INCREMENT BY 1
    MINVALUE 1
    MAXVALUE 9999999999
    NO CYCLE
    CACHE 20;
```

## 使用序列

```sql
INSERT INTO users (id, username) VALUES (user_id_seq.NEXTVAL, 'alice');
SELECT user_id_seq.NEXTVAL FROM DUMMY;
SELECT user_id_seq.CURRVAL FROM DUMMY;
```

## 修改序列

```sql
ALTER SEQUENCE user_id_seq RESTART WITH 1000;
ALTER SEQUENCE user_id_seq INCREMENT BY 2;
```

## 删除序列

```sql
DROP SEQUENCE user_id_seq;
DROP SEQUENCE user_id_seq CASCADE;
```

## IDENTITY 列

## GENERATED ALWAYS AS IDENTITY

```sql
CREATE TABLE users (
    id       BIGINT GENERATED ALWAYS AS IDENTITY,
    username NVARCHAR(64) NOT NULL,
    email    NVARCHAR(255) NOT NULL
);
```

## GENERATED BY DEFAULT AS IDENTITY

```sql
CREATE TABLE orders (
    id       BIGINT GENERATED BY DEFAULT AS IDENTITY,
    amount   DECIMAL(10,2)
);
```

## 带参数的 IDENTITY

```sql
CREATE TABLE products (
    id       BIGINT GENERATED BY DEFAULT AS IDENTITY (START WITH 100 INCREMENT BY 1),
    name     NVARCHAR(64)
);
```

## UUID 生成

```sql
SELECT SYSUUID FROM DUMMY;                   -- 返回 32 位十六进制字符串
SELECT NEWUID() FROM DUMMY;                  -- SAP HANA 2.0+，标准 UUID 格式

CREATE TABLE sessions (
    id         NVARCHAR(36) DEFAULT NEWUID(),
    user_id    BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 序列 vs 自增 权衡

## IDENTITY：简单，一张表一个

## SEQUENCE：灵活，可跨表共享

## SYSUUID / NEWUID()：全局唯一

## SAP HANA 是内存数据库，序列性能极佳
