## SEQUENCE

```sql
CREATE SEQUENCE user_id_seq
    START WITH 1
    INCREMENT BY 1
    MINVALUE 1
    MAXVALUE 9223372036854775807
    CACHE 250000                            -- 默认 250000，MPP 环境中高缓存很重要
    NO CYCLE;
```


使用序列
```sql
INSERT INTO users (id, username) VALUES (NEXTVAL('user_id_seq'), 'alice');
SELECT NEXTVAL('user_id_seq');
SELECT CURRVAL('user_id_seq');
```


修改序列
```sql
ALTER SEQUENCE user_id_seq RESTART WITH 1000;
ALTER SEQUENCE user_id_seq CACHE 500000;
```


删除序列
```sql
DROP SEQUENCE user_id_seq;
DROP SEQUENCE IF EXISTS user_id_seq;
```


## AUTO_INCREMENT / IDENTITY

```sql
CREATE TABLE users (
    id       BIGINT AUTO_INCREMENT,          -- 或 IDENTITY
    username VARCHAR(64) NOT NULL,
    email    VARCHAR(255) NOT NULL
);
```


IDENTITY 带参数
```sql
CREATE TABLE orders (
    id       BIGINT IDENTITY(1, 1),          -- IDENTITY(start, step)
    amount   DECIMAL(10,2)
);
```


## GENERATED AS IDENTITY（SQL 标准）

```sql
CREATE TABLE products (
    id       BIGINT GENERATED BY DEFAULT AS IDENTITY (START WITH 100 INCREMENT BY 1),
    name     VARCHAR(64)
);

CREATE TABLE logs (
    id       BIGINT GENERATED ALWAYS AS IDENTITY,
    message  VARCHAR(1000)
);
```


## UUID 生成

```sql
SELECT UUID_GENERATE();                      -- 返回 BINARY(16)
```

转为字符串显示
SELECT TO_HEX(UUID_GENERATE());

## 序列 vs 自增 权衡

1. AUTO_INCREMENT/IDENTITY：简单，表级
2. SEQUENCE：灵活，可跨表共享，CACHE 大值提升 MPP 性能
3. GENERATED AS IDENTITY：SQL 标准兼容
4. UUID_GENERATE()：全局唯一
5. Vertica MPP 架构中，高 CACHE 值对序列性能至关重要
6. AUTO_INCREMENT 在分布式加载时可能不连续

限制：
SERIAL / BIGSERIAL 不支持
UUID_GENERATE() 返回二进制，需要转换
高并发加载时 AUTO_INCREMENT 值不连续
