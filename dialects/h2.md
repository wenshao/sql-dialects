# H2

**分类**: Java 嵌入式数据库
**文件数**: 51 个 SQL 文件
**总行数**: 4015 行

## 概述与定位

H2 是一个用纯 Java 编写的开源关系型数据库，由 Thomas Mueller（HSQLDB 的原作者）开发。H2 的核心定位是轻量级嵌入式数据库，特别适合 Java 应用的单元测试、原型开发和嵌入式场景。它同时支持嵌入模式（应用进程内运行）和服务器模式（独立进程），支持内存数据库和磁盘持久化。H2 的一大亮点是提供多种兼容模式——可以模拟 MySQL、PostgreSQL、Oracle、SQL Server 等数据库的行为。

## 历史与演进

- **2004 年**：Thomas Mueller 启动 H2 项目，目标改进 HSQLDB 的设计。
- **2005 年**：H2 1.0 发布，提供嵌入式和服务器两种模式。
- **2011 年**：引入 MVCC 存储引擎提升并发性能。
- **2019 年**：2.0 版本开始大规模重构，引入新存储引擎（MVStore）。
- **2021 年**：2.x 版本稳定化，改进兼容模式和 SQL 标准支持。
- **2023-2025 年**：持续维护更新，保持作为 Java 生态首选嵌入式数据库的地位。

## 核心设计思路

H2 的设计追求**极简和高效**：整个数据库打包为一个约 2.5 MB 的 JAR 文件。存储引擎使用 MVStore（Multi-Version Store），一种基于日志结构的 B-Tree 变体，支持 MVCC 并发控制。H2 提供**多兼容模式**——通过 `MODE=MySQL`、`MODE=PostgreSQL`、`MODE=Oracle`、`MODE=MSSQLServer` 等参数切换，调整 SQL 解析和函数行为以模拟目标数据库。内置 Web 控制台提供浏览器交互界面。

## 独特特色

- **多兼容模式**：`jdbc:h2:mem:test;MODE=MySQL` 启动时指定兼容模式，模拟不同数据库的 SQL 行为。
- **内存 + 文件双模式**：`jdbc:h2:mem:` 纯内存数据库用于测试，`jdbc:h2:file:./data` 持久化到磁盘。
- **CSVREAD/CSVWRITE**：`SELECT * FROM CSVREAD('data.csv')` 直接查询 CSV 文件作为表。
- **LINK TABLE**：`CREATE LINKED TABLE` 连接外部数据库（通过 JDBC）实现联邦查询。
- **单文件部署**：整个数据库引擎是一个 JAR 文件，零外部依赖。
- **Web 控制台**：内置 H2 Console（Web 界面），无需安装客户端工具。
- **全文搜索**：内置基于 Lucene 或原生实现的全文索引。

## 已知不足

- 不适合生产环境的高并发 OLTP 场景——并发写入性能有限。
- 兼容模式虽覆盖常见场景但并非完美，复杂 SQL 可能表现不一致。
- 单文件存储引擎的最大数据量受限（TB 级数据场景不适用）。
- 缺乏生产级的备份恢复、复制和高可用方案。
- 社区规模小，主要由少数核心开发者维护。
- 存储过程和触发器使用 Java 类实现，与标准 SQL 过程语言差异大。

## 对引擎开发者的参考价值

H2 是学习数据库内核实现的理想参考——纯 Java 单代码库实现了完整的 SQL 数据库（解析器、优化器、存储引擎、事务管理），代码可读性好。多兼容模式的实现展示了如何在同一引擎中通过配置切换 SQL 方言行为（函数映射、类型转换规则、分页语法等）。MVStore 存储引擎的设计也是一个清晰的 MVCC B-Tree 实现范例。CSVREAD 展示了如何将外部文件透明映射为关系表。
