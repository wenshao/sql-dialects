# SQL 标准: INSERT

> 参考资料:
> - [ISO/IEC 9075 SQL Standard](https://www.iso.org/standard/76583.html)
> - [Modern SQL - by Markus Winand](https://modern-sql.com/)
> - [Modern SQL - INSERT](https://modern-sql.com/feature/insert)

## SQL-89 (SQL1)

单行插入（最基础的 INSERT 语法）
```sql
INSERT INTO users (username, email, age) VALUES ('alice', 'alice@example.com', 25);
```

从查询结果插入
```sql
INSERT INTO users_archive (username, email, age)
SELECT username, email, age FROM users WHERE age > 60;
```

省略列名（按表定义顺序，不推荐）
```sql
INSERT INTO users VALUES ('alice', 'alice@example.com', 25);
```

## SQL-92 (SQL2)

指定默认值
```sql
INSERT INTO users (username, email, age) VALUES ('alice', 'alice@example.com', DEFAULT);
```

DEFAULT VALUES（所有列使用默认值）
```sql
INSERT INTO users DEFAULT VALUES;
```

子查询作为值
```sql
INSERT INTO users (username, email, age)
VALUES ('alice', (SELECT email FROM temp WHERE id = 1), 25);
```

## SQL:1999 (SQL3)

行值构造器可用于 INSERT
```sql
INSERT INTO users (username, email, age) VALUES ('alice', 'alice@example.com', 25);
```

## SQL:2003

多行 VALUES（多行插入标准化）
```sql
INSERT INTO users (username, email, age) VALUES
    ('alice', 'alice@example.com', 25),
    ('bob', 'bob@example.com', 30),
    ('charlie', 'charlie@example.com', 35);
```

IDENTITY 列（自动生成值）
GENERATED ALWAYS AS IDENTITY
GENERATED BY DEFAULT AS IDENTITY

OVERRIDING 子句（覆盖 IDENTITY 列的自动值）
```sql
INSERT INTO users (id, username, email)
OVERRIDING SYSTEM VALUE
VALUES (100, 'alice', 'alice@example.com');
```

```sql
INSERT INTO users (id, username, email)
OVERRIDING USER VALUE
VALUES (100, 'alice', 'alice@example.com');
```

OVERRIDING USER VALUE 忽略用户提供的值，使用系统生成值

## SQL:2008

MERGE 语句（见 upsert）

## SQL:2011

时态表支持
INSERT 到系统版本化表时自动记录时间戳

## SQL:2016

JSON 数据类型可直接插入
INSERT INTO docs (id, data) VALUES (1, JSON '{"name": "alice"}');

## 各版本差异总结

SQL-89:  基本 INSERT INTO ... VALUES / INSERT INTO ... SELECT
SQL-92:  DEFAULT VALUES, DEFAULT 关键字
SQL:1999: 行值构造器
SQL:2003: 多行 VALUES, IDENTITY, OVERRIDING
SQL:2008: MERGE 语句标准化
SQL:2011: 时态表
SQL:2016: JSON 支持
