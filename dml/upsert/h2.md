## MERGE INTO ... KEY（H2 简化语法）


## 基本 MERGE（最常用的 UPSERT 方式）

```sql
MERGE INTO users (id, username, email) KEY (id)
VALUES (1, 'alice', 'alice@example.com');
```

## 多行 MERGE

```sql
MERGE INTO users (id, username, email, age) KEY (id)
VALUES (1, 'alice', 'alice@example.com', 25),
       (2, 'bob', 'bob@example.com', 30);
```

## KEY 可以是任意唯一列

```sql
MERGE INTO users (id, username, email) KEY (username)
VALUES (1, 'alice', 'alice_new@example.com');
```

## MERGE（SQL 标准语法）


```sql
MERGE INTO users AS t
USING (SELECT 1 AS id, 'alice' AS username, 'alice@example.com' AS email) AS s
ON t.id = s.id
WHEN MATCHED THEN
    UPDATE SET username = s.username, email = s.email
WHEN NOT MATCHED THEN
    INSERT (id, username, email) VALUES (s.id, s.username, s.email);
```

## 从表中 MERGE

```sql
MERGE INTO users AS t
USING staging_users AS s
ON t.id = s.id
WHEN MATCHED THEN
    UPDATE SET username = s.username, email = s.email
WHEN NOT MATCHED THEN
    INSERT (id, username, email) VALUES (s.id, s.username, s.email);
```

## 条件 MERGE

```sql
MERGE INTO users AS t
USING staging_users AS s
ON t.id = s.id
WHEN MATCHED AND s.age > t.age THEN
    UPDATE SET age = s.age
WHEN MATCHED AND s.age <= t.age THEN
    DELETE
WHEN NOT MATCHED THEN
    INSERT (id, username, email, age) VALUES (s.id, s.username, s.email, s.age);
```

## INSERT ... ON DUPLICATE KEY UPDATE（MySQL 兼容模式）


在 MySQL 兼容模式下可用
SET MODE MYSQL;
INSERT INTO users (id, username, email) VALUES (1, 'alice', 'alice@example.com')
ON DUPLICATE KEY UPDATE username = VALUES(username), email = VALUES(email);

## REPLACE INTO（MySQL 兼容模式）


在 MySQL 兼容模式下可用
REPLACE INTO users (id, username, email) VALUES (1, 'alice', 'alice@example.com');
注意：MERGE INTO ... KEY 是 H2 特有的简化 UPSERT
注意：也支持标准 SQL MERGE 语法
注意：MySQL 兼容模式下支持 ON DUPLICATE KEY UPDATE 和 REPLACE
注意：KEY 列必须有唯一索引
