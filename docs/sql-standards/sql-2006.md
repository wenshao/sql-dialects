# SQL:2006 标准

SQL:2006（ISO 9075:2006）是一个相对较小的版本更新，主要集中在 XML 功能的大幅增强上，特别是将 XQuery 语言集成到了 SQL 中。由于 JSON 的兴起，XML 相关特性在新引擎中的采纳率很低，因此这个版本对现代引擎开发者的影响有限。

## 核心变化：SQL/XML 增强

### XQuery 集成

SQL:2006 最重要的变化是将 W3C 的 XQuery 1.0 集成到 SQL/XML（ISO 9075-14）部分：

```sql
-- XMLQUERY: 在 SQL 中执行 XQuery 表达式
SELECT XMLQUERY(
    'for $e in /employees/employee
     where $e/salary > 50000
     return $e/name'
    PASSING xml_data
    RETURNING CONTENT
) FROM company_data;

-- XMLTABLE: 将 XML 数据转换为关系表（最实用的函数）
SELECT x.*
FROM company_data,
     XMLTABLE(
         '/employees/employee'
         PASSING xml_data
         COLUMNS
             name    VARCHAR(100) PATH 'name',
             salary  DECIMAL(10,2) PATH 'salary',
             dept    VARCHAR(50)  PATH 'department'
     ) AS x;

-- XMLEXISTS: 检查 XQuery 表达式是否返回非空结果
SELECT * FROM documents
WHERE XMLEXISTS('/doc/section[@id="intro"]' PASSING xml_column);
```

### XML 类型转换增强

```sql
-- XMLCAST: XML 值与 SQL 值之间的类型转换
SELECT XMLCAST(XMLQUERY('/price' PASSING data) AS DECIMAL(10,2))
FROM products;

-- XMLPARSE: 从字符串创建 XML 值
INSERT INTO docs (xml_col) VALUES (XMLPARSE(DOCUMENT '<root><item>test</item></root>'));

-- XMLSERIALIZE: 将 XML 值转换为字符串
SELECT XMLSERIALIZE(DOCUMENT xml_col AS VARCHAR(4000)) FROM docs;
```

## 各引擎 XML 支持情况

| 特性 | PostgreSQL | Oracle | SQL Server | MySQL | SQLite | BigQuery | Snowflake | ClickHouse |
|------|-----------|--------|-----------|-------|--------|----------|-----------|------------|
| XML 数据类型 | ✓ | ✓ (XMLTYPE) | ✓ (xml) | ✗ | ✗ | ✗ | ✗ | ✗ |
| XMLQUERY | ✓ (xpath) | ✓ | ✓ (.query) | ✗ | ✗ | ✗ | ✗ | ✗ |
| XMLTABLE | ✓ (10+) | ✓ | ✓ (OPENXML) | ✓ (ExtractValue, 有限) | ✗ | ✗ | ✗ | ✗ |
| XMLEXISTS | ✓ (xpath_exists) | ✓ | ✓ (.exist) | ✗ | ✗ | ✗ | ✗ | ✗ |
| XQuery 完整支持 | 部分 | ✓ | ✓ | ✗ | ✗ | ✗ | ✗ | ✗ |

关键差异：
- **Oracle** 对 XML 支持最完整，紧跟标准，有原生 XMLTYPE 存储
- **SQL Server** 有完整的 XML 支持但使用非标准方法名（`.query()`、`.value()`、`.exist()`、`.nodes()`）
- **PostgreSQL** 支持 XML 类型和 xpath 函数，10+ 版本支持标准的 XMLTABLE
- **MySQL** 只有非常有限的 `ExtractValue()` 和 `UpdateXML()` 函数，已被标记为弃用
- **新一代引擎**（BigQuery、Snowflake、ClickHouse）完全不支持 XML，转向 JSON

## XMLTABLE 的实际价值

尽管 SQL:2006 整体影响有限，XMLTABLE 是一个值得关注的特性。它的设计模式后来直接影响了 SQL:2016 中 JSON_TABLE 的设计：

```sql
-- XMLTABLE 模式（SQL:2006）
SELECT x.* FROM data,
    XMLTABLE('/root/item' PASSING xml_col
        COLUMNS name VARCHAR(100) PATH 'name') AS x;

-- JSON_TABLE 模式（SQL:2016）—— 几乎相同的设计
SELECT j.* FROM data,
    JSON_TABLE(json_col, '$.items[*]'
        COLUMNS (name VARCHAR(100) PATH '$.name')) AS j;
```

这个设计模式（将半结构化数据展平为关系表）证明了其价值，只是载体从 XML 换成了 JSON。

## 其他次要变更

- **SQL 例程中的 XQuery 支持**：存储过程和函数中可以使用 XQuery
- **XML 索引增强**：标准化了 XML 数据的索引策略（各引擎实现差异很大）
- **SQL/XML 符合性要求更新**：细化了 SQL/XML 合规测试

## 对引擎开发者的实现建议

1. **XML 支持优先级低**。除非你的目标用户群体是传统企业（金融、医疗、政府），否则不建议投入 XML 支持。
2. **如果需要 XML，只实现 XMLTABLE**。它是最实用的功能——将 XML 文档解析为行和列，方便与关系数据集成。
3. **将精力投入 JSON**。SQL:2016 的 JSON 支持在设计上吸收了 XML 的教训，且市场需求远大于 XML。如果已经实现了 JSON_TABLE，XMLTABLE 的架构可以复用。
4. **理解 XMLTABLE → JSON_TABLE 的演化关系**。如果你打算实现 JSON_TABLE（强烈推荐），了解 XMLTABLE 的设计有助于理解标准的设计意图。
