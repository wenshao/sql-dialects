## 注意: Firebird 5.0 之前没有原生 JSON 类型，JSON 存储为 VARCHAR/BLOB

## json_value 提取标量字段（Firebird 5.0+）


```sql
SELECT id,
       json_value(data, '$.customer')               AS customer,
       CAST(json_value(data, '$.total') AS DOUBLE)   AS total,
       json_value(data, '$.address.city')            AS city,
       json_value(data, '$.address.zip')             AS zip
FROM   orders_json;
```

## json_value: SQL/JSON 标准函数，提取标量值

支持JSONPath 语法（如 $.address.city 访问嵌套对象）

## json_query 提取 JSON 片段


```sql
SELECT id,
       json_query(data, '$.items') AS items_json
FROM   orders_json;
```

json_query: 提取 JSON 对象或数组（返回字符串）
与 json_value 的区别:
json_value 返回标量（字符串、数字）
json_query 返回 JSON 片段（对象、数组）

## json_table 展开数组为关系行（Firebird 5.0+）


```sql
SELECT o.id,
       o.data,
       jt.product,
       CAST(jt.qty AS INT)       AS qty,
       CAST(jt.price AS DOUBLE)  AS price
FROM   orders_json o,
       JSON_TABLE(o.data, '$.items[*]'
           COLUMNS (
               product VARCHAR(100) PATH '$.product',
               qty     VARCHAR(10)  PATH '$.qty',
               price   VARCHAR(10)  PATH '$.price'
           )
       ) AS jt;
```

JSON_TABLE 是 SQL/JSON 标准的核心函数
一步完成 JSON 数组展开 + 列映射
类似于 MySQL 8.0 的 JSON_TABLE 和 Oracle 的 JSON_TABLE

## 使用 EXECUTE BLOCK 模拟复杂展平（Firebird 4.0 回退方案）


对于 Firebird 4.0（无 JSON 函数），使用 PSQL 循环解析
EXECUTE BLOCK RETURNS (id INT, customer VARCHAR(100))
AS
DECLARE VARIABLE json_data VARCHAR(10000);
DECLARE VARIABLE pos_start INTEGER;
DECLARE VARIABLE pos_end INTEGER;
BEGIN
FOR SELECT o.id, o.data FROM orders_json o INTO :id, :json_data DO
BEGIN
pos_start = POSITION('"customer":' IN :json_data) + 12;
pos_end   = POSITION('"', :json_data, pos_start + 1);
customer  = SUBSTRING(:json_data FROM :pos_start FOR :pos_end - :pos_start);
SUSPEND;
END
END
注意: 字符串解析方式非常脆弱，仅适用于受控的简单场景
强烈建议升级到 Firebird 5.0 使用原生 JSON 函数

## Firebird JSON 支持演进:

Firebird 4.0: 无 JSON 支持，需字符串函数或 UDF
Firebird 5.0: json_value, json_query, json_table（SQL/JSON 标准）
2. 与其他数据库对比:
PostgreSQL: LATERAL + jsonb_array_elements（组合式）
MySQL:      JSON_TABLE（声明式，8.0+）
Firebird:   JSON_TABLE（声明式，5.0+）
Oracle:     JSON_TABLE（声明式，12c+）
对引擎开发者:
SQL/JSON 标准的 JSON_TABLE 是最用户友好的展平方案
Firebird 5.0 选择兼容 SQL/JSON 标准而非自创语法是正确决策
对于旧版本，PSQL 存储过程是实用的回退方案
