## 关键差异

- **标识符引号**是迁移中最容易忽略的差异：MySQL/Hive/Spark 用反引号 `` ` ``，PostgreSQL/Oracle/标准 SQL 用双引号 `"`，SQL Server 还支持方括号 `[]`
- **字符串连接**：Oracle/PostgreSQL 用 `||`，SQL Server 用 `+`，MySQL 必须用 `CONCAT()` 函数
- **NULL 处理**：MySQL 用 `IFNULL`，Oracle 用 `NVL`，SQL Server 用 `ISNULL`，通用写法是 `COALESCE`
- **Oracle 空串等于 NULL**，这是从其他方言迁移到 Oracle 时最大的语义陷阱
- **Oracle 的 CONCAT 只接受 2 个参数**，多参数拼接必须用 `||` 操作符
- **MySQL 不支持 FULL OUTER JOIN**，需要 `LEFT JOIN UNION RIGHT JOIN` 模拟
- **自增列**语法差异巨大：MySQL 用 `AUTO_INCREMENT`，PostgreSQL 用 `SERIAL` 或 `IDENTITY`，Oracle 12c 前需 `SEQUENCE + TRIGGER`
- **日期格式化**的格式符完全不同：MySQL 用 `%Y-%m-%d`，PostgreSQL/Oracle 用 `YYYY-MM-DD`，SQL Server 用数字代码
- **OceanBase / DamengDB / KingbaseES / PolarDB** 等支持兼容模式，可降低迁移成本
- **分布式/NewSQL** 数据库通常兼容某种传统方言（MySQL 或 PostgreSQL），迁移时要注意不完全兼容的边界
- **大数据引擎**通常不支持事务性 DML（UPDATE/DELETE），SCD 等场景需要改用 INSERT OVERWRITE 模式
