# Databricks: 迁移速查表 (Migration Cheatsheet)

> 参考资料:
> - [Databricks SQL Reference](https://docs.databricks.com/sql/language-manual/index.html)
> - [Databricks Migration Guide](https://docs.databricks.com/migration/index.html)


一、数据类型: 兼容 Spark SQL + Delta Lake
INT→INT, BIGINT→BIGINT/LONG, FLOAT→FLOAT, DOUBLE→DOUBLE,
VARCHAR→STRING, DECIMAL→DECIMAL(p,s), BOOLEAN→BOOLEAN,
DATE→DATE, TIMESTAMP→TIMESTAMP, BINARY→BINARY,
JSON→STRING(用from_json), ARRAY→ARRAY<T>, MAP→MAP<K,V>, STRUCT→STRUCT<...>
二、函数: 与Spark SQL完全兼容, 额外支持Delta Lake特有函数
三、陷阱: Delta Lake是默认表格式, Unity Catalog管理权限,
AUTO_INCREMENT→GENERATED ALWAYS AS IDENTITY(Delta Lake),
支持MERGE INTO(很强大), 支持CLONE, 支持Time Travel
四、自增: GENERATED ALWAYS AS IDENTITY 或 BIGINT GENERATED BY DEFAULT AS IDENTITY
五、日期: current_timestamp(); current_date();
date_add(d,1); datediff(a,b); date_format(ts,'yyyy-MM-dd HH:mm:ss')
to_date(s, 'yyyy-MM-dd'); to_timestamp(s, 'yyyy-MM-dd HH:mm:ss')
from_unixtime(epoch); unix_timestamp(ts)
year(d); month(d); day(d)
六、字符串: length, upper, lower, trim, substring, regexp_replace, locate, concat, concat_ws

## 七、数据类型映射（从 MySQL/PostgreSQL/Hive 到 Databricks）

MySQL → Databricks:
- INT → INT, BIGINT → BIGINT/LONG, FLOAT → FLOAT,
- DOUBLE → DOUBLE, DECIMAL(p,s) → DECIMAL(p,s),
- VARCHAR(n) → STRING, TEXT → STRING,
- DATETIME → TIMESTAMP, DATE → DATE,
- BOOLEAN → BOOLEAN, BLOB → BINARY,
- JSON → STRING (用from_json/to_json),
- AUTO_INCREMENT → GENERATED ALWAYS AS IDENTITY
PostgreSQL → Databricks:
- INTEGER → INT, TEXT → STRING, SERIAL → IDENTITY,
- BOOLEAN → BOOLEAN, JSONB → STRING,
- ARRAY → ARRAY<T>, BYTEA → BINARY
Hive → Databricks: 高度兼容
- STRING → STRING, MAP → MAP, ARRAY → ARRAY,
- STRUCT → STRUCT


### 八、函数等价映射

MySQL → Databricks:
- IFNULL → IFNULL/COALESCE, NOW() → CURRENT_TIMESTAMP(),
- DATE_FORMAT → DATE_FORMAT (格式码不同!),
- CONCAT → CONCAT, GROUP_CONCAT → COLLECT_LIST + CONCAT_WS,
- LIMIT → LIMIT


### 九、常见陷阱补充

Delta Lake 是默认表格式（支持 ACID 事务）
Unity Catalog 管理权限和数据治理
MERGE INTO 非常强大（CDC/SCD/UPSERT）
支持 CLONE (深浅克隆)
支持 Time Travel (版本回溯)
OPTIMIZE 命令合并小文件
Z-ORDER 优化数据布局
VACUUM 清理过期文件
Photon 引擎加速查询


### 十、NULL 处理

IFNULL(a, b); COALESCE(a, b, c);
NULLIF(a, b); NVL(a, b); NVL2(a, b, c);


### 十一、分页语法

SELECT * FROM t ORDER BY id LIMIT 10 OFFSET 20;


### 十二、日期格式码 (Java SimpleDateFormat)

yyyy=年, MM=月, dd=日, HH=24时, mm=分, ss=秒
> **注意**: 与 MySQL (%Y/%m/%d) 格式码不同!
