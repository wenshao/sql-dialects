# Firebird: 迁移速查表 (Migration Cheatsheet)

> 参考资料:
> - [Firebird SQL Reference](https://firebirdsql.org/file/documentation/html/en/refdocs/fblangref40/)


一、数据类型: 标准SQL类型
INT→INTEGER, BIGINT→BIGINT, FLOAT→FLOAT, DOUBLE→DOUBLE PRECISION,
VARCHAR→VARCHAR(n), TEXT→BLOB SUB_TYPE TEXT, DECIMAL→DECIMAL(p,s),
BOOLEAN→BOOLEAN(3.0+), DATE→DATE, TIMESTAMP→TIMESTAMP,
BLOB→BLOB, AUTO_INCREMENT→GENERATED BY DEFAULT AS IDENTITY(3.0+)
二、陷阱: BLOB SUB_TYPE TEXT用于大文本, SEQUENCE(生成器)用于自增,
存储过程用PSQL语法, 字符集需显式指定, 页大小影响行大小限制
三、自增: GENERATED BY DEFAULT AS IDENTITY 或 SEQUENCE

```sql
CREATE SEQUENCE my_seq;
```

## 六、数据类型映射（从 MySQL/PostgreSQL/Oracle 到 Firebird）

MySQL → Firebird:
- INT → INTEGER, BIGINT → BIGINT, FLOAT → FLOAT,
- DOUBLE → DOUBLE PRECISION, VARCHAR(n) → VARCHAR(n),
- TEXT → BLOB SUB_TYPE TEXT, DECIMAL(p,s) → DECIMAL(p,s),
- DATETIME → TIMESTAMP, DATE → DATE,
- BOOLEAN → BOOLEAN (3.0+), BLOB → BLOB,
- AUTO_INCREMENT → GENERATED BY DEFAULT AS IDENTITY (3.0+),
- TINYINT → SMALLINT, JSON → 不支持, ENUM → VARCHAR+CHECK
PostgreSQL → Firebird:
- INTEGER → INTEGER, TEXT → BLOB SUB_TYPE TEXT,
- SERIAL → GENERATED BY DEFAULT AS IDENTITY,
- BOOLEAN → BOOLEAN (3.0+), JSONB → 不支持,
- ARRAY → 不支持, BYTEA → BLOB
Oracle → Firebird:
- NUMBER → NUMERIC, VARCHAR2 → VARCHAR,
- CLOB → BLOB SUB_TYPE TEXT, SEQUENCE → SEQUENCE,
- SYSDATE → CURRENT_TIMESTAMP

### 七、函数等价映射

MySQL → Firebird:
- IFNULL → COALESCE, NOW() → CURRENT_TIMESTAMP,
- CONCAT(a,b) → a || b,
- DATE_FORMAT → EXTRACT + 手动拼接,
- GROUP_CONCAT → LIST (Firebird 2.1+),
- LIMIT → FIRST/SKIP 或 OFFSET/FETCH (3.0+)

### 八、常见陷阱补充

BLOB SUB_TYPE TEXT 用于大文本（替代 TEXT/CLOB）
字符集需显式指定 (推荐 UTF8)
页大小影响行大小限制 (4096/8192/16384/32768)
PSQL 存储过程语法与其他数据库不同
- **SEQUENCE (生成器) 语法**: GEN_ID(seq_name, 1) 或 NEXT VALUE FOR
无内置日期/数值格式化函数

### 九、NULL 处理

COALESCE(a, b, c);
NULLIF(a, b);
IIF(a IS NULL, b, a);                              -- Firebird 3.0+

### 十、分页语法

- **Firebird 3.0+**: SELECT * FROM t ORDER BY id OFFSET 20 ROWS FETCH NEXT 10 ROWS ONLY;
- **旧版**: SELECT FIRST 10 SKIP 20 * FROM t ORDER BY id;
