# Firebird: Date/Time Types

> 参考资料:
> - [Firebird SQL Reference](https://firebirdsql.org/en/reference-manuals/)
> - [Firebird Release Notes](https://firebirdsql.org/file/documentation/release_notes/html/en/4_0/rlsnotes40.html)


DATE: date only, 4 bytes
TIME: time of day, 4 bytes (default 0 fractional seconds, up to 4)
TIMESTAMP: date + time, 8 bytes (default 0 fractional seconds, up to 4)
4.0+: TIME WITH TIME ZONE, TIMESTAMP WITH TIME ZONE

```sql
CREATE TABLE events (
    id         BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    event_date DATE,
    event_time TIME,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 4.0+: Time zone types

```sql
CREATE TABLE tz_events (
    local_time  TIME,
    tz_time     TIME WITH TIME ZONE,
    local_ts    TIMESTAMP,
    tz_ts       TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

## Current date/time

```sql
SELECT CURRENT_DATE FROM RDB$DATABASE;           -- DATE
SELECT CURRENT_TIME FROM RDB$DATABASE;            -- TIME
SELECT CURRENT_TIMESTAMP FROM RDB$DATABASE;       -- TIMESTAMP
SELECT 'NOW' FROM RDB$DATABASE;                   -- context variable, returns TIMESTAMP
SELECT LOCALTIME FROM RDB$DATABASE;               -- TIME (without time zone)
SELECT LOCALTIMESTAMP FROM RDB$DATABASE;           -- TIMESTAMP (without time zone)
```

## Date arithmetic

```sql
SELECT CURRENT_DATE + 7 FROM RDB$DATABASE;                   -- add 7 days
SELECT CURRENT_DATE - 30 FROM RDB$DATABASE;                  -- subtract 30 days
SELECT DATEADD(MONTH, 3, CURRENT_DATE) FROM RDB$DATABASE;    -- add 3 months
SELECT DATEADD(YEAR, 1, CURRENT_DATE) FROM RDB$DATABASE;     -- add 1 year
SELECT DATEADD(HOUR, 2, CURRENT_TIMESTAMP) FROM RDB$DATABASE;
SELECT DATEADD(MINUTE, 30, CURRENT_TIMESTAMP) FROM RDB$DATABASE;
```

## Date difference

```sql
SELECT DATEDIFF(DAY, DATE '2024-01-01', DATE '2024-12-31') FROM RDB$DATABASE;   -- 365
SELECT DATEDIFF(MONTH, DATE '2024-01-01', DATE '2024-06-01') FROM RDB$DATABASE;
SELECT DATEDIFF(YEAR, DATE '2020-01-01', DATE '2024-01-01') FROM RDB$DATABASE;
SELECT DATEDIFF(HOUR, TIMESTAMP '2024-01-01 00:00:00', TIMESTAMP '2024-01-01 12:00:00') FROM RDB$DATABASE;
```

## Extract

```sql
SELECT EXTRACT(YEAR FROM CURRENT_DATE) FROM RDB$DATABASE;
SELECT EXTRACT(MONTH FROM CURRENT_DATE) FROM RDB$DATABASE;
SELECT EXTRACT(DAY FROM CURRENT_DATE) FROM RDB$DATABASE;
SELECT EXTRACT(HOUR FROM CURRENT_TIMESTAMP) FROM RDB$DATABASE;
SELECT EXTRACT(MINUTE FROM CURRENT_TIMESTAMP) FROM RDB$DATABASE;
SELECT EXTRACT(SECOND FROM CURRENT_TIMESTAMP) FROM RDB$DATABASE;
SELECT EXTRACT(WEEKDAY FROM CURRENT_DATE) FROM RDB$DATABASE;  -- 0=Sunday
SELECT EXTRACT(YEARDAY FROM CURRENT_DATE) FROM RDB$DATABASE;  -- day of year
SELECT EXTRACT(WEEK FROM CURRENT_DATE) FROM RDB$DATABASE;     -- ISO week (4.0+)
```

## Date literals

```sql
SELECT DATE '2024-01-15' FROM RDB$DATABASE;
SELECT TIME '10:30:00' FROM RDB$DATABASE;
SELECT TIMESTAMP '2024-01-15 10:30:00' FROM RDB$DATABASE;
```

## Cast strings to dates

```sql
SELECT CAST('2024-01-15' AS DATE) FROM RDB$DATABASE;
SELECT CAST('2024-01-15 10:30:00' AS TIMESTAMP) FROM RDB$DATABASE;
```

## 4.0+: time zone conversion

```sql
SELECT CURRENT_TIMESTAMP AT TIME ZONE 'America/New_York' FROM RDB$DATABASE;
SELECT CURRENT_TIMESTAMP AT LOCAL FROM RDB$DATABASE;
```

Note: DATEADD and DATEDIFF are the primary date arithmetic functions
Note: date +/- integer adds/subtracts days
Note: RDB$DATABASE is the single-row system table
Note: 4.0+ added time zone support and WEEK extraction
Note: fractional seconds precision is up to 4 digits (10000ths of second)
