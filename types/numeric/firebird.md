# Firebird: Numeric Types

> 参考资料:
> - [Firebird SQL Reference](https://firebirdsql.org/en/reference-manuals/)
> - [Firebird Release Notes](https://firebirdsql.org/file/documentation/release_notes/html/en/4_0/rlsnotes40.html)


Integer types
SMALLINT:  2 bytes, -32768 ~ 32767
INTEGER:   4 bytes, -2^31 ~ 2^31-1
BIGINT:    8 bytes, -2^63 ~ 2^63-1 (1.5+)

```sql
CREATE TABLE examples (
    small_val  SMALLINT,
    int_val    INTEGER,
    big_val    BIGINT
);
```

No TINYINT or UNSIGNED types
Decimal / Numeric (exact)
DECIMAL(p,s) / NUMERIC(p,s): precision up to 18 (34 in 4.0+ with DECFLOAT)
Internally stored as BIGINT with implicit scale

```sql
CREATE TABLE prices (
    price     DECIMAL(10,2),          -- up to 99999999.99
    rate      NUMERIC(18,6),          -- max precision 18
    quantity  DECIMAL(9,0)
);
```

Floating point
FLOAT: 4 bytes, ~7 digits precision
DOUBLE PRECISION: 8 bytes, ~15 digits precision

```sql
CREATE TABLE measurements (
    float_val  FLOAT,
    double_val DOUBLE PRECISION
);
```

## Boolean (3.0+)

```sql
CREATE TABLE flags (
    active BOOLEAN DEFAULT FALSE
);
```

## INT128 (4.0+, 16-byte integer)

```sql
CREATE TABLE large_numbers (
    huge_val INT128                    -- -2^127 ~ 2^127-1
);
```

DECFLOAT (4.0+, decimal floating-point)
DECFLOAT(16): 16-digit precision
DECFLOAT(34): 34-digit precision

```sql
CREATE TABLE financial (
    amount DECFLOAT(34),
    rate   DECFLOAT(16)
);
```

## Auto-increment: Generators / Sequences

Generator (legacy syntax)

```sql
CREATE GENERATOR gen_users_id;
CREATE SEQUENCE seq_users;
```

## Identity columns (3.0+)

```sql
CREATE TABLE auto_id (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name VARCHAR(100)
);
```

## Domains for numeric types

```sql
CREATE DOMAIN d_money AS DECIMAL(12,2)
    DEFAULT 0.00
    CHECK (VALUE >= 0);

CREATE DOMAIN d_percentage AS DECIMAL(5,2)
    CHECK (VALUE BETWEEN 0 AND 100);
```

Note: DECIMAL/NUMERIC max precision is 18 (stored in BIGINT)
Note: 4.0+ adds INT128 and DECFLOAT for larger numbers
Note: FLOAT is 32-bit (unlike Teradata where FLOAT is always 64-bit)
Note: GEN_ID(generator, increment) returns next value
Note: NEXT VALUE FOR sequence is the modern syntax (3.0+)
